My revisions and commentary

In the example code provided my MDN there isn't any CSS defined for the class="content." In my version of the code I added the class and set a max-width of 80 em. I did this within the media query for screens with a min-width of 70em. The purpose of this paragraph describes what I did. And it allows me to see the effect of the change by limiting the max-width of the content area on larger screens.

Looking at the page given the changes that I made: it appears to me that the content area is too wide. It allows at least 20 average sized words per line. I believe that most UX designers recommend about 12 words per line, maximum. It helps with readability. So I'm going to modify the @media query parameters as well as the content area width to see if I can achieve a better line width for UX design.

I achieved the width that I wanted by adding CSS for a class named content and setting the max-width value. But, in doing so I broke the behavior of the rest of the page. It seems that it is always displaying in a three-column layout. Here is my code and the MDN original code for comparison.
