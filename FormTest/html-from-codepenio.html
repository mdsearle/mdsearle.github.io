My First Form?

December 18, 2020

This isn't my first HTML form. I've created lots of forms; none of them looked very nice. None of them performed a real function. All of them were "test" forms where nothing was submitted to a server.

I'm trying to reboot my web development learning by using focused guides to help direct my study and development. This page and form are based on the Your First Form guide on MDN.

I'm also hoping to learn some "best practices" and web design standards along the way. To that end, I'm trying to incorporate the principles advocated by Stéphanie Walter in her article UX And HTML5: Let’s Help Users Fill In Your Mobile Form (Part 1) published on the Smashing Magazine website.

Using the HTML and CSS code copied from the MDN article, this is what the form looks like. Note that the Submit button doesn't work properly because I haven't created the page pointed to by the form action. And, at the moment, this page isn't getting served by a web server.

The code provided by MDN doesn't follow the guidelines/recommendations provided by Stéphanie. So I will attempt to recode the form to more closely match her recommendations. The revised form follows.




I made the following changes to make the form more mobile friendly.

  • Moved the labels above the fields to allow more width for the fields on the limited mobile device screen width
  • Reduced the width of the form to 290px to accommodate the narrower screen width of almost all mobile devices. However, the form still overflows the width of the screen on an iPhone 5 with a screen width of 320px. I am a little disappointed by this finding. I want to play with @media queries, max-width, and device-width to see whether I can find the correct combination of CSS settings to get the form onto the iPhone 5 screen.
  • Increased the height of the input fields to 30px to make the hit zone a little larger. I think it also improves the look of the spacing on the form.
